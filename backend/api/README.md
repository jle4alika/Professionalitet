# API — Сервис аренды вендинговых аппаратов

REST API на FastAPI для управления пользователями, заказами, платежами и вендинговыми аппаратами.

## Запуск

```bash
# Из корня проекта
uvicorn backend.api.main:app --reload --host 0.0.0.0 --port 8000
```

Документация: **http://localhost:8000/docs**

## Структура

```
api/
├── main.py              # Точка входа FastAPI
├── dependencies/        # Зависимости (db_session)
├── routers/             # Роутеры API
│   ├── user.py          # Пользователи
│   ├── order.py         # Заказы
│   ├── payment.py       # Платежи
│   ├── rent.py          # Аренда
│   └── vending_machine.py # Вендинговые аппараты
└── schemas/             # Pydantic-схемы (DTO)
    ├── user.py
    ├── order.py
    ├── payment.py
    ├── vending_machine.py
    └── common.py        # Общие схемы ответов
```

## Base URL

Все эндпоинты доступны по префиксу **`/api/v1`**.

## Эндпоинты

### Пользователи (`/api/v1/users`)

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/login` | Авторизация |
| POST | `/logout` | Выход |
| GET | `/auth` | Проверка авторизации (JWT) |
| POST | `/registration` | Регистрация |
| GET | `/` | Список пользователей |
| GET | `/{id}` | Пользователь по ID |
| PATCH | `/{id}` | Обновление |
| DELETE | `/{id}` | Удаление |

### Заказы (`/api/v1/orders`)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Список заказов (фильтры: user_id, expired) |
| GET | `/{id}` | Заказ по ID |
| POST | `/` | Создание заказа |
| PATCH | `/{id}` | Обновление (продление, expired) |
| DELETE | `/{id}` | Удаление/отмена |

### Платежи (`/api/v1/payments`)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Список платежей (фильтры: user_id, order_id, success) |
| GET | `/{id}` | Платёж по ID |
| GET | `/{id}/check` | Проверка статуса |
| POST | `/` | Создание платежа |
| PATCH | `/{id}` | Обновление статуса |

### Аренда (`/api/v1/rents`)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Активные аренды |
| GET | `/available` | Свободные аппараты |
| POST | `/` | Создание аренды |

### Вендинговые аппараты (`/api/v1/vending-machines`)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Список (фильтр: rented) |
| GET | `/{id}` | Аппарат по ID |
| POST | `/` | Создание |
| PATCH | `/{id}` | Обновление |
| DELETE | `/{id}` | Удаление |

## Пагинация

Для списков доступны query-параметры:
- `skip` — пропустить записей (по умолчанию 0)
- `limit` — максимум записей (по умолчанию 100)

## Авторизация

JWT передаётся в cookies. После `/login` или `/registration` токен устанавливается автоматически. Для `/auth` требуется авторизованный запрос.

## Тесты

```bash
# Из корня проекта
pytest backend/tests/ -v
```

По умолчанию используется SQLite in-memory. Для PostgreSQL задайте `TEST_DATABASE_URL` в окружении.
